# Bible Reader Psychology UX Redesign

目的：重新審視目前聖經閱讀器的 UI/UX，將產品重心從「完成、統計、成就」調整為「經文、領受、默想」。本文件作為後續設計與實作依據，優先處理左側目錄、右側功能、上方設定與成就系統。

## 1. 設計判斷

目前產品已經有良好的基礎：紙感背景、serif 閱讀字體、深淺色主題、章節導航、筆記、每日靈修與搜尋。但互動模型同時引入了 streak、完成章節、成就解鎖、閱讀統計、熱圖、每日目標與倒數，容易把聖經閱讀變成任務管理。

從心理學角度，這會帶來三個問題：

- 讀經動機從內在動機轉向外在獎勵，長期可能降低默想品質。
- streak、達標、倒數、成就會放大 guilt loop：沒讀時感到虧欠，進而更不想打開。
- 介面焦點被「我完成多少」分散，而不是「這段經文正在對我說什麼」。

設計目標不是完全取消紀錄，而是把紀錄降級為安靜的回顧，把主要空間還給經文、領受與默想。

## 2. 核心原則

1. 經文優先
   閱讀頁預設只呈現經文、章節定位、簡潔導航與默想入口。所有非閱讀必要資訊預設收合。

2. 低壓陪伴
   避免「達標、連勝、勇士、解鎖、倒數」等績效語言。改用「最近停留、今日領受、默想紀錄、閱讀回顧」。

3. 右側作為默想室
   右側不應再是統計面板優先，而是「領受、默想、筆記、禱告」的支援區。

4. 左側只做定位
   左側目錄只負責幫助使用者找到經文，不負責成就、今日進度、streak 或任務提醒。

5. 上方設定只放環境控制
   上方工具列只保留會影響閱讀狀態的設定，例如字級、主題、沉浸模式、搜尋、面板開關。

## 3. 移除或降級成就系統

### 建議移除

- `AchievementModal`
- 成就解鎖彈窗
- `first_chapter`、`streak_7`、`streak_30`、`century` 等競賽式命名
- 「百章勇士」「連續 30 天」這類身份化獎勵
- 主流程中的 emoji 慶祝，例如完成書卷時大幅干擾閱讀

### 建議降級保留

可以保留 localStorage 裡的完成紀錄，但不要用成就包裝。改為「閱讀回顧」：

- 最近讀過
- 曾停留的書卷
- 已完成的書卷，使用低彩度標記
- 本週閱讀痕跡，預設收合

### 文案替換

| 目前語言 | 建議語言 |
| --- | --- |
| 成就解鎖 | 閱讀回顧已更新 |
| 連續 7 天 | 最近一週有閱讀 |
| 今日達標 | 今天已有停留 |
| 完成本章 | 標記已讀 |
| 百章勇士 | 已讀 100 章 |
| 閱讀統計 | 閱讀回顧 |

## 4. 左側目錄重設

### 現況問題

`Sidebar` 目前同時包含：

- streak / 已讀章數 / 今日目標
- 書卷搜尋
- 舊約 / 新約 / 雅煞珥書
- 章節完成進度條
- 底部搜尋、筆記、靈修工具列

這讓左欄承載太多角色。使用者想找章節時，也會看到進度與任務訊號。

### 新設計

左側只保留「經文定位」：

- 搜尋書卷
- 舊約、新約、雅煞珥書
- 當前書卷展開
- 當前章節高亮
- 已讀章節使用非常弱的紙色標記，不出現綠色勾勾或進度條

### 移除

- 左欄頂部 stats row
- streak emoji
- 已讀總章數
- 今日 x/y
- 書卷完成進度條
- 章節完成的強烈綠色 ring 與 checkmark
- 左欄底部三等分工具列

### 替代

左欄底部只放一個安靜入口：

- `最近停留：馬太福音 5`
- 點擊回到上次閱讀

如需搜尋、筆記、靈修，移到上方工具列或右側面板。

## 5. 右側功能重設

### 右側面板定位

右側應成為「默想輔助區」，不是 dashboard。建議改成一個 `ReflectionPanel`，內含三個 tab：

- 領受
- 默想
- 筆記

### 領受 tab

功能目的：幫助使用者從經文進入今日主題。

內容：

- 今日靈修引用經文
- 一句核心問題
- 跳到相關經文
- 禱告文

避免：

- 過多折疊段落
- 外部連結比經文本身更醒目
- 每日任務感文案

### 默想 tab

功能目的：讓使用者慢下來。

建議提供固定三個 prompt：

- 這段經文中，我注意到什麼？
- 這段經文觸動了我哪個處境？
- 今天我想帶走的一句話是什麼？

這些 prompt 不要強迫填寫，也不要顯示完成狀態。

### 筆記 tab

功能目的：回顧過去 highlight 與筆記。

建議：

- 依最近時間排序
- 顯示經文引用、短句、個人筆記
- 顏色標記改成柔和紙色，不用紅、藍、亮黃
- 點擊可跳回經文

### 閱讀回顧入口

`StatsDashboard` 不應作為主要右側功能。建議改名為 `ReadingReviewPanel`，放在設定或次層選單中。

保留內容：

- 最近閱讀
- 書卷回顧
- 閱讀計畫設定

降級內容：

- heatmap 預設收合
- streak 不放第一屏
- 成就區移除

## 6. 上方設定重設

上方工具列應只放閱讀環境與全域操作。

建議排列：

- 左：目錄開關、目前位置，例如 `馬太福音 5`
- 中：閱讀進度細線，只顯示本章 scroll progress，不顯示完成率
- 右：搜尋、字級、主題、沉浸模式、默想面板

設定面板內容：

- 字級：小 / 中 / 大
- 行距：舒適 / 寬鬆
- 主題：紙白 / 夜讀
- 節號：顯示 / 淡化 / 隱藏
- 筆記提示：開 / 關

避免把閱讀計畫、成就、streak 放進上方主工具列。

## 7. 閱讀頁內容呈現

### 經文排版

目前 `Reader` 的閱讀寬度與行高方向正確，可再調整：

- 桌面最大寬度維持 680-720px。
- 行高維持 1.85-1.95。
- 字距不要超過 `0.01em`，避免中文閱讀鬆散。
- 節號透明度再降低，或提供設定可隱藏。
- 章標題應清楚但不浮誇，避免被工具列取代。

### 完成本章按鈕

目前完成按鈕位於章節導航中間，會暗示「讀經的終點是打勾」。建議改為：

- 預設不顯示。
- 滾到章末後，以文字連結呈現：`標記這章已讀`。
- 點擊後只做低干擾提示：`已記錄這次閱讀`。

### Resume CTA

可保留，但文案降低工具感：

- 目前：繼續上次閱讀
- 建議：回到上次停留

## 8. 顏色與材質

目前紙色方向是對的，應繼續避免科技感、霓虹、玻璃、強漸層。

建議色彩：

- 背景：`#F8F4EA` 或 `#F5F2EC`
- 閱讀紙面：`#FDFBF7`
- 主文字：`#26231F`
- 次文字：`#6B6258`
- 邊框：`#E4DCCF`
- 靜態標記：`#D8C89F`
- 主要操作：`#4F7358`
- 警示或刪除：只在真正破壞性操作使用，避免出現在 highlight 常用色

Highlight 顏色建議改為語義而非彩虹：

- 金：重要
- 綠：安慰
- 灰藍：疑問
- 赭：禱告/回應

材質：

- 保留極淡紙張 noise。
- 卡片只用於筆記、靈修段落、回顧列表。
- 不要巢狀卡片。
- 右側面板可像書籤紙條，不要像 analytics dashboard。

## 9. 內容語氣

整體文案應像陪伴，不像教練或績效系統。

建議語氣：

- `回到上次停留`
- `今天想默想什麼？`
- `寫下一句領受`
- `已記錄這次閱讀`
- `閱讀回顧`

避免語氣：

- `達標`
- `衝刺`
- `連勝`
- `解鎖`
- `勇士`
- `還差 x 章`
- `倒數`

## 10. 建議實作階段

### Phase 1: 降低干擾

- 移除 `AchievementModal` 的顯示流程。
- `StatsDashboard` 中移除成就區。
- `Sidebar` 移除頂部 stats row。
- 左側章節完成樣式降級。
- 完成提示改成低干擾 toast 或文字。

### Phase 2: 重組資訊架構

- 新增 `ReflectionPanel`。
- 將 `DailyDevotional` 與 `NotesPanel` 合併為右側默想輔助區。
- 搜尋移到上方工具列。
- 閱讀回顧移到設定或次層入口。

### Phase 3: 閱讀體驗細修

- 新增節號顯示設定。
- 新增行距設定。
- 調整 highlight 語義色。
- 重新設計章末操作，讓 `標記已讀` 成為次要動作。

## 11. 驗收標準

完成後的第一屏應符合：

- 使用者打開 app，第一注意力落在經文，不是進度或工具。
- 左側只回答「我要去哪一章」。
- 右側只回答「我如何領受與默想」。
- 上方只回答「我如何調整閱讀環境」。
- 沒有成就彈窗、連續天數壓力、每日倒數或排行榜式文案。
- 使用者即使很多天沒讀，再打開也不會被提醒失敗，只會看到可繼續停留的位置。
